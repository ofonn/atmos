 Atmos — AI-Powered Weather Web App

 Context

 Building an agentic AI weather platform as a responsive web app. The app combines standard weather functionality (current conditions, forecasts,
 metrics) with a conversational AI assistant powered by Gemini that answers context-aware questions like "How should I dress today?" or "Should I
 wear a hoodie?" The visual direction is a clean, modern mobile-first design with blue/purple gradients, card-based layout, and Poppins typography
 (from reference images).

 ---
 Tech Stack

 - Framework: Next.js 14 (App Router) — SSR, API routes to hide keys, easy deployment
 - Styling: Tailwind CSS + custom design tokens matching reference palette
 - AI: Google Gemini API (@google/generative-ai)
 - Weather: OpenWeatherMap API (current, forecast, UV, alerts)
 - Icons: React Icons + custom weather icon set (3D-style SVGs or Lucide)
 - Notifications: Web Push API with Service Workers
 - State: React Context + SWR for data fetching/caching
 - Deployment: Vercel (natural fit for Next.js)

 ---
 Design System (from references)

 - Fonts: Poppins (primary), Inter (fallback)
 - Colors:

   - Dark: #1A1C26

 - Style: Glassmorphism cards, gradient headers, rounded corners, soft shadows
 - Layout: Mobile-first responsive, max-width container for desktop

 ---
 Implementation Plan

 Phase 1: Project Setup & Skeleton

 1. Initialize Next.js 14 project with TypeScript + Tailwind
 2. Set up design tokens (colors, fonts, spacing) in tailwind.config.ts
 3. Create root layout with Poppins font import
 4. Set up .env.local template for API keys
 5. Create basic page shell with gradient header

 Phase 2: Weather Data Layer

 1. Create TypeScript types for weather data (types/weather.ts)
 2. Build OpenWeatherMap API client (lib/weather.ts)
   - getCurrentWeather(lat, lon) — current conditions
   - getForecast(lat, lon) — 5-day/3-hour forecast
   - getOneCall(lat, lon) — hourly + daily + alerts + UV (One Call API 3.0)
 3. Create API routes to proxy weather requests (keeps API key server-side)
   - GET /api/weather?lat=X&lon=X → current weather
   - GET /api/forecast?lat=X&lon=X → forecast data
 4. Build useLocation hook (browser geolocation + city search)
 5. Build useWeather hook (SWR-based fetching + caching)

 Phase 3: Weather UI Components

 1. Header — Location display, hamburger menu, notification bell
 2. CurrentWeather — Hero card with large temp, weather icon, condition, feels-like, hi/lo. Blue/purple gradient background matching references.
 3. WeatherMetrics — Grid of 4 cards: rain chance, precipitation, wind speed, UV index
 4. HourlyForecast — Horizontal scrollable cards with time, icon, temp. Highlight current hour.
 5. DailyForecast — 7-day list with day, icon, hi/lo temps, condition
 6. SunriseSunset — Visual timeline showing sunrise/sunset times
 7. WeatherIcon — Map weather condition codes to appropriate icons

 Phase 4: AI Chat Assistant

 1. Build Gemini client (lib/gemini.ts) with a weather-focused system prompt:
   - "You are Atmos, an AI weather assistant. You have access to the user's current weather data and forecast. Answer questions about what to wear,
  weather safety, outdoor planning, etc. Be conversational, helpful, and specific."
 2. Create streaming chat API route (/api/chat)
   - Accepts user message + current weather context as input
   - Streams Gemini response back
   - Injects real-time weather data into the prompt so AI gives accurate answers
 3. Build chat UI components:
   - ChatPanel — Slide-up panel from bottom (or side panel on desktop)
   - ChatMessage — Styled message bubbles (user vs AI)
   - ChatInput — Text input with send button
   - QuickPrompts — Chips: "What should I wear?", "Should I bring an umbrella?", "Is it safe to go running?", "What's the UV like today?"
 4. Build useChat hook — manages conversation history, streaming, loading states

 Phase 5: Notifications

 1. Set up Service Worker (public/sw.js)
 2. Create notification permission request flow
 3. Build notification triggers:
   - Severe weather alerts (from OpenWeatherMap alerts data)
   - Daily morning summary ("Today: 72°F, sunny — light jacket recommended")
   - Rain warning ("Rain expected at 3 PM — bring an umbrella!")
 4. Build useNotifications hook

 Phase 6: Polish & Responsive Design

 1. Mobile-first responsive layout (looks like the reference images on mobile, expands nicely on desktop)
 2. Loading skeletons for all data sections
 3. Error states and offline fallback
 4. Smooth animations/transitions (Framer Motion for chat panel, card reveals)
 5. Dark mode support (the references show a darker variant)

 ---
 AI System Prompt Strategy

 The Gemini AI will receive a structured system prompt containing:
 You are Atmos, a friendly and knowledgeable AI weather assistant.

 CURRENT WEATHER DATA:
 - Location: {city}, {country}
 - Temperature: {temp}°F (feels like {feels_like}°F)
 - Condition: {description}
 - Humidity: {humidity}%
 - Wind: {wind_speed} mph
 - UV Index: {uvi}
 - Sunrise: {sunrise}, Sunset: {sunset}

 HOURLY FORECAST (next 12 hours):
 {hourly_data}

 DAILY FORECAST (next 7 days):
 {daily_data}

 INSTRUCTIONS:
 - Give practical, specific advice based on the actual weather data above
 - For clothing questions, consider: temperature, wind chill, rain chance, UV
 - For safety questions, flag severe weather, high UV, extreme temps
 - Be conversational and concise
 - If asked about times to watch, highlight significant weather changes

 This ensures the AI always answers with real, current data — not hallucinated weather.
